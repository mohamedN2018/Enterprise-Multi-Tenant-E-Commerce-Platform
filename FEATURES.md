## 📌 نظرة عامة

**q-shop** ليست مجرد متجر إلكتروني — إنها **بنية تحتية كاملة لسوق تجاري متعدد البائعين (Multi-Vendor Marketplace)**. تسمح المنصة لأي عدد من التجار بإنشاء متاجرهم الخاصة، وإدارة منتجاتهم ومخزونهم وطلباتهم بشكل مستقل تمامًا، بينما يشرف مدير المنصة على الجميع من لوحة تحكم مركزية واحدة.

بُنيت المنصة على أساس **العزل الكامل بين المتاجر (True Multi-Tenancy)** — كل متجر هو حدود صارمة لبياناته: عملاؤه، منتجاته، سلاته، وطلباته منفصلة تمامًا عن أي متجر آخر، مع فرض العزل على مستوى قاعدة البيانات.

> 🎯 **الفكرة باختصار:** منصة SaaS جاهزة لإطلاق سوق إلكتروني مثل Amazon أو نون أو Shopify، بمعايير هندسية على مستوى المؤسسات.

## ✨ المميزات الكاملة

### 🛒 تجربة العميل (Storefront)

- **صفحة رئيسية احترافية** — قسم Hero تفاعلي، الأكثر مبيعًا، التصنيفات، أحدث المنتجات، والعروض.
- **تصفّح ذكي للمنتجات** — بحث فوري، فلترة، ترتيب، وتقسيم صفحات (Pagination).
- **صفحة منتج غنية** — معرض صور، خيارات ومتغيرات المنتج (المقاس/اللون)، تقييمات، ومنتجات مقترحة.
- **سلة تسوّق كاملة** مع حجز فوري للمخزون ومنع الشراء بما يفوق الكمية المتاحة.
- **إتمام شراء متكامل (Checkout)** — عناوين، طرق شحن، حساب الضريبة، الكوبونات، والدفع.
- **تتبّع الطلبات** — دورة حياة كاملة للطلب مع خط زمني (Timeline) وتأكيد التسليم.
- **حساب العميل** — الملف الشخصي، سجل الطلبات، دفتر العناوين، وقائمة الأمنيات (Wishlist).
- **نظام تقييمات موثّق** — لا يمكن تقييم المنتج إلا بعد **شرائه واستلامه فعليًا** (Verified Purchase).
- **متاجر بعلامات تجارية** — لكل بائع صفحة متجر خاصة بعنوان (Slug) مميز.
- **تصميم متجاوب بالكامل** — تجربة مثالية على الجوال والحاسب، مع Header احترافي للموبايل.

### 🏪 لوحة تحكم البائع (Seller Console)

- **لوحة معلومات (Dashboard)** بمخططات بيانية حيّة (ApexCharts) لأداء المتجر.
- **إدارة الكتالوج** — منتجات، تصنيفات، علامات تجارية، خصائص، ومتغيرات.
- **منتجات رقمية** — أصول قابلة للتحميل، مفاتيح تراخيص (License Keys)، ومنح تحميل.
- **حزم المنتجات (Bundles)** — بيع عدة منتجات كمجموعة واحدة.
- **إدارة المخزون متعدد المستودعات** — استلام، تعديل، ونقل المخزون، مع سجل حركات كامل.
- **إدارة الطلبات** — تحديث الحالة، تعيين شركة الشحن، ورقم التتبّع.
- **العروض والتسويق** — كوبونات خصم، حملات تسويقية، وبطاقات هدايا (Gift Cards).
- **التسعير المتقدم** — مجموعات عملاء وقواعد تسعير مخصّصة.
- **الشحن** — مناطق شحن وطرق شحن بحساب تلقائي للتكلفة.
- **المرتجعات (RMA)** — استقبال طلبات الإرجاع، الموافقة/الرفض، واسترداد المبالغ.
- **المدفوعات والتحصيلات (Payouts)** — دفتر أستاذ مالي وعمولة منصة قابلة للضبط.
- **التقارير المالية** — الضرائب، العملات المتعددة، والإيرادات.
- **إدارة الفريق** — دعوة موظفين وتحديد **صلاحيات دقيقة لكل قسم** (Area Permissions).
- **إعدادات المتجر** — بما فيها تكامل نظام الكاشير.

### 🖥️ لوحة تحكم مدير المنصة (Super-Admin)

- **إشراف مركزي** على جميع المتاجر والبائعين في السوق.
- **الموافقة على طلبات البائعين** — زيادة حدود عدد المتاجر (Seller Limits).
- **تخصيص هوية المنصة بالكامل** — تغيير الألوان الأساسية والثانوية، لون الخلفية، والخطوط، مع **قوالب جاهزة (Presets)** — للمدير فقط، وتُطبَّق على الموقع مباشرةً.
- **مراقبة الاحتيال** — مراجعة عمليات الفحص وقرارات المخاطر.
- **إدارة كل شيء** — الوصول لكل متجر بصلاحيات المالك للإدارة والدعم.

### 🌐 مميزات السوق (Marketplace)

- **نموذج عمولة** — احتساب عمولة المنصة تلقائيًا على مبيعات كل بائع.
- **نظام تحصيلات البائعين** — دفتر أستاذ (Ledger) وتسويات مالية.
- **دليل المتاجر** — صفحة تجمع كل متاجر السوق للعملاء.
- **برنامج ولاء متكامل** — نقاط ولاء، محفظة رصيد المتجر (Wallet)، وبرنامج إحالة (Referral).

### 🔌 تكامل نظام الكاشير (POS Integration)

- **تكامل ثنائي الاتجاه** مع أنظمة نقاط البيع الخارجية (مثل q-shop POS):
  - **استيراد (Outbound):** ربط مورّد POS واستيراد كتالوج منتجاته تلقائيًا.
  - **دفع (Inbound):** إصدار مفتاح API يتيح للكاشير قراءة المخزون وتسجيل المبيعات.
- **مزامنة ذكية** — دفع الطلبات الأونلاين إلى سجل الكاشير، والتحقق من توفر المخزون قبل الدفع.
- **مفاتيح مشفّرة** عند التخزين + مزامنة عبر Celery في الخلفية.

### 📦 سلسلة التوريد والتصنيع (Procurement)

- **إدارة الموردين** وأوامر الشراء (Purchase Orders) مع الاستلام.
- **تتبّع الدفعات (Batches)** بتواريخ انتهاء صلاحية + فحص آلي للدفعات المنتهية.
- **الأرقام التسلسلية (Serial Numbers)** لتتبّع الوحدات.
- **التصنيع الخفيف** — قوائم المواد (Bill of Materials) وأوامر التشغيل (Work Orders).

## 🏗️ التميّز الهندسي (Architecture)

- **🔒 عزل متعدد المستأجرين (Multi-Tenancy):** كل صف بيانات مرتبط بمتجره، ويُفرض العزل تلقائيًا عبر Middleware يحلّ المتجر النشط (`X-Store-Id`) ومديري استعلام مُقيّدين (Scoped Managers) — استحالة تسرّب بيانات بين المتاجر.

- **🧱 معمارية نظيفة متعددة الطبقات:** فصل واضح `Models → Repositories → Services → Views`. منطق العمل كله في **طبقة خدمات (Service Layer)**، وواجهات رفيعة.

- **📜 عقد موحّد للنماذج:** مفاتيح UUID، طوابع زمنية، حذف ناعم (Soft Delete)، وطوابع تدقيق (Audit) على كل نموذج.

- **📦 غلاف استجابة موحّد (Response Envelope):** كل استجابات الـ API بصيغة `{success, message, data, errors, error_code}` — وتجربة تكامل متسقة.

- **🎯 معالجة أخطاء احترافية:** هرمية أخطاء عمل (Domain Errors) تُترجَم لأكواد وأكواد HTTP صحيحة.

- **🔍 قابلية الرصد (Observability):** تتبّع كل طلب بمعرّف فريد (Request-ID) في السجلّات المهيكلة.

## 🛡️ الأمان والجودة

- ✅ **مصادقة JWT** مع تدوير التوكِن وقائمة سوداء وخاصية "تذكّرني".
- ✅ **تجزئة Argon2** لكلمات المرور (الأقوى حاليًا).

- ✅ **تحديد معدّل الطلبات (Throttling)** لكل نقطة حساسة (تسجيل دخول، تسجيل، استعادة كلمة مرور، ربط POS).
- ✅ **حماية من SSRF** عند الاتصال بالخدمات الخارجية.
- ✅ **صلاحيات دقيقة (RBAC)** — أدوار مالك/مدير/موظف بصلاحيات لكل قسم.
- ✅ **إدارة جلسات الأجهزة** — عرض الأجهزة المتصلة وإلغاء الوصول.
- ✅ **+451 اختبار آلي** يغطّي منطق العمل الحرج.
- ✅ **توثيق OpenAPI** تلقائي لكل نقاط الـ API.

## 🌍 التوطين (Localization)

- **العربية هي اللغة الأساسية**، والإنجليزية احتياطية.
- **دعم RTL كامل** — الاتجاه يتبدّل تلقائيًا مع اللغة.
- **ترجمة من الخادم أيضًا** — بيانات المنتجات والمتاجر تُترجم عبر ترويسة `Accept-Language`.
- **الجنيه المصري (EGP)** كعملة افتراضية، مع دعم عملات متعددة.
- **خطوط عربية محلية** (Cairo) بدون أي اعتماد على روابط خارجية.
